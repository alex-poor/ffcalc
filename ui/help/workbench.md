# Workbench

Model a bulk-funding offer for one practice. Everything updates live.

## Using this screen

1. **Move the pass-through sliders** to set the % of each stream you're offering the practice. The PHO retains the remainder.
2. **Save as scenario** to keep these slider positions for this practice.
3. **Save as template** (in the Templates card) if you'll reuse this slider pattern across other practices.

## Templates vs scenarios

- A **scenario** is tied to *this practice*. Use it to model "Practice X under deal A vs deal B". Scenarios appear in **Open comparison**.
- A **template** is reusable across practices. Apply one to load its sliders; save the current sliders as a template to reuse them elsewhere.

## Why First-Level is locked at 100%

First-Level capitation passes through entirely by default — it's the bulk of the patient-attached funding. To model retention on it, click **Unlock** in the locked row (opens Settings → Advanced).

U14, U6 are always 100% pass-through.

## Variance vs current

Shows how your offer compares to the practice's current PHO offer. No variance? The practice has no baseline saved. Add one via **Edit practice → Current PHO offer**.

## Connections

- **Edit practice** — fix demographics or baseline.
- **Open comparison** — view this practice's scenarios side-by-side.
- **Network** (sidebar) — apply one pass-through across all practices.
